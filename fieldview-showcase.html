§01 The thesis

Three commitments that make this hard to build honestly

A normal CRM asks the user. FieldView forbids the question — which forces real inference everywhere, and a conscience to match.

01 — THE MOAT

Zero SE data entry

Required fields are forbidden in the product. Every signal is derived from passively captured activity — there is no "fill in the deal health" form to fall back on. The VP sees value before any sales engineer logs in.

Consequence: real inference wherever a CRM would just ask.
02 — THE LOOP

The product grades itself

It records every prediction, watches what the operator did, watches what happened to the deal, then retunes its own thresholds weekly based on whether it was right. A closed learning loop, not a static rules engine.

Mechanism: predict → grade → calibrate, on a cron.
03 — THE CONSCIENCE

It refuses to lie

When data is thin it renders "insufficient sample — N of M outcomes collected" instead of a screenshot-able percentage. Every rollup is gated behind operator confirmation, so the model never trains on its own guesses.

Enforced by append-only ledgers + CI tests.

§05 The loop

The moat. The product doesn't just emit predictions — it closes the loop on every one, and a confirmation gate keeps auto-graded guesses out of the training set.

The clever grade: correct_but_mitigated

A pessimistic prediction (AT_RISK, STALLED…) on a deal that then won looks "wrong." But if the operator completed a follow-up inside the mitigation window and the health score rose ≥10 points, the system credits the warning as correct_but_mitigated — the alarm was right, and acting on it is why the outcome flipped.

accuracy = (correct + correct_but_mitigated) / judgeable

§09 Under the hood

Engineering rigor, for the skeptics

Multi-tenancy is absolute
All 84 models carry orgId; every query filters on it; raw SQL is banned; a static-analysis test grep-walks every server action and fails CI if any query omits orgId.
Encryption at rest
All OAuth tokens (Salesforce, Google, Slack) are AES-256-GCM encrypted; safeDecrypt throws if the key is unset in prod.
Server actions, not API routes
Every mutation calls requireSession / requireAdmin / requireOwner. API routes exist only for OAuth callbacks, cron, share links, uploads, PDF download.
Hydration discipline
No wall-clock/random in client render; every toLocale* pins timeZone:UTC — each backed by a regression test, born from a real production freeze.
